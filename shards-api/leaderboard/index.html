Leaderboard API

info

Partners can also use self-integration portal to create/update leaderboards

Create a New Leaderboard​

Endpoint​

POST /v1/leader-boards

Authorization​

  • Guarded by ApiKeyGuard
  • Requires Api key (api-key)

Headers​

  • api-key: <api_key>

Description​

This endpoint is used to create a new leaderboard for a guild. It requires information about the guild, the type of leaderboard, and additional metadata, if applicable.

Request Body​

The request body must conform to the CreateLeaderBoardDto structure and should include the following fields:

  • name (string): The name of the guild.
  • type (string): The type of the leaderboard, either 'increased' or 'replace'.
  • metadata (object): Additional metadata about the guild, optional.
  • clientId (string): The identifier of the game client.

Example of a valid request payload:

{
  "name": "cat",
  "type": "increased",
  "metadata": {
    "description": "This is a cat",
    "image": "https://i.imgur.com/4M34hi2.jpeg"
  },
  "clientId": "a0bac604-0fa4-447a-a3de-4deff02008c4"
}

Response​

Upon successful creation, the API will return a LeaderBoards object.

{
  "status": "Success",
  "data": {
    // The LeaderBoards object will be returned here
  }
}

Update Guild Score​

Endpoint​

PUT /v1/guild-score

Authorization​

  • Guarded by ApiKeyGuard
  • Requires Api key (api-key)

Headers​

  • api-key: <api_key>

Description​

This endpoint allows for updating the score of a specific guild. It requires details about the guild, the leaderboard, and the score to be updated.

Request Body​

The request body must follow the UpdateGuildScoreDto structure and should include the following fields:

  • guildId (string): The identifier of the guild whose score is to be updated.
  • leaderBoardId (string): The identifier of the leaderboard associated with this score.
  • clientId (string): The identifier of the client making the request.
  • score (number): The new score to be updated for the guild.

Example of a valid request payload:

{
  "guildId": "5f9d7a3c9d5f1d0b7c9bce0e",
  "leaderBoardId": "5f9d7a3c9d5f1d0b7c9bce0e",
  "clientId": "5f9d7a3c9d5f1d0b7c9bce0e",
  "score": 100
}

Response​

200 OK​

Upon the successful update, the API will return a GuildScore object.

{
  "status": "Success",
  "data": {
    // The updated GuildScore object
  }
}
