Game Configuration API

info

Partners can also use self-integration portal for this configuration

Create New Game Configuration​

Endpoint​

POST /v1/game-config

Authorization​

  • Guarded by ApiKeyGuard
  • Requires Api key (api-key)

Headers​

  • api-key: <api_key>

Description​

Allows the creation of a new game configuration. This endpoint is intended for administrative use within a ShardsTech server to set up new game configurations.

Request Body​

The request body should be a JSON object with the following fields:

{
  "linkDapp": "The link Dapp of the game (optional)",
  "callBack": "The user id of the guild (optional)",
  "webHook": "The webhook URL of the game (optional)",
  "gameServerUrl": "The game server URL of the game (optional)",
  "authorizationGameServerUrl": "The authorization URL for the game server (optional)",
  "fee": "The fee amount (optional)",
  "allowUpdateFields": ["Fields allowed to be updated (optional)"],
  "name": "The name of the game (optional)",
  "metadata": "The metadata of the game (optional)",
  "memberGuildConfig": "The member guild config (optional)",
  "leaderBoardId": "The leaderboard ID (optional)",
  "sendNotificationHashKey": "The hash key for sending notifications (optional)",
  "clientId": "The ID of the game client",
  "chainIds": "The chains id of the chain",
  "endpoint": "The endpoint of the game (optional)",
  "clientApiKey": "The client api key of the game (optional)",
  "authMethod": "Game config authentication method, values: MiraiId/Other",
  "gamePayCreateGuild": "Does game pay create guild fee (optional)",
  "useGuildInShards": "Does use guild in Shards (optional)",
  "logo": "The avatar of the game (optional)",
  "enableDisbandGuild": "Is enable disband guild (optional)",
  "supportWebBrowser": "Is support web browser (optional)",
  "status": "The status of the game (optional)",
  "background": "The background of the game"
}

Response​

Upon successful creation, the API will return a GameConfig object.

{
  "status": "Success",
  "data": {
    // The GameConfig object
  }
}

Tags​

Admin ShardsTech

Update Existing Game Configuration​

Endpoint​

PUT /v1/game-config

Authorization​

  • Guarded by ApiKeyGuard
  • Requires Api key (api-key)

Headers​

  • api-key: <api_key>

Description​

Updates an existing game configuration. This endpoint is intended for administrative use to modify game configurations already set up in the ShardsTech server.

Request Body​

The request body should include the UpdateGameConfigDto object with the fields necessary for updating the game configuration.

{

  "callBack": "The user id of the guild (optional)",

  "leaderBoardId": "The leaderboard ID (optional)",
  "sendNotificationHashKey": "The hash key for sending notifications (optional)",
  "clientId": "The ID of the game client",
  "chainIds": "The chains id of the chain",
  "endpoint": "The endpoint of the game (optional)",
  "clientApiKey": "The client api key of the game (optional)",
  "authMethod": "Game config authentication method, values: MiraiId/Other",
  "gamePayCreateGuild": "Does game pay create guild fee (optional)",
  "useGuildInShards": "Does use guild in Shards (optional)",
