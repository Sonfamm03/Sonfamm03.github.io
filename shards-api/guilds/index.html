Guilds API

Config Number of Allowed Updates for Multiple Guilds​

Endpoint​

PUT /v1/guilds/allow-update

Authorization​

  • Guarded by ApiKeyGuard
  • Requires Bearer token api-key

Headers​

  • Authorization: Bearer api_key

Description​

This API is used to config the number of allowed updates for multiple guilds. This endpoint is designed to be used by game servers to manage update allowances across different guilds.

Request Body​

The request body must be JSON with the following structure:

{
  "guildIds": ["array of guild identifiers"],
  "clientId": "client identifier",
  "numberAllowUpdate": "number of allowed updates",
  "startAllowUpdateTimestamp": "timestamp marking the start of the update allowance period",
  "endAllowUpdateTimestamp": "timestamp marking the end of the update allowance period",
  "isAll": "boolean indicating if the allowance applies to all guilds"
}

Parameters:

  • guildIds: An array of guild identifiers that the updates apply to.
  • clientId: The identifier for the client making the request.
  • numberAllowUpdate: The number of updates that are allowed.
  • startAllowUpdateTimestamp: The starting timestamp from when the updates are allowed.
  • endAllowUpdateTimestamp: The ending timestamp until when the updates are allowed.
  • isAll: A boolean flag indicating whether the update allowance should apply to all guilds.

Response​

The update was successful.

Update Guild Metadata​

Endpoint​

PATCH /v1/guilds/:id

Authorization​

  • Guarded by ApiKeyGuard
  • Requires Api key (api-key)

Headers​

  • api-key: <api_key>

Description​

This endpoint is used to update the metadata for a specific guild. It is utilized by game servers to adjust guild settings as needed.

URL Parameters​

  • id: Unique identifier of the guild to be updated.

Request Body​

The request should contain the following fields as part of the UpdateGuildDto:

  • metadata: An object containing the new metadata for the guild.
  • clientId: A string representing the client identifier.

The request body must adhere to the structure defined by the UpdateGuildDto. Here is an example of a valid request payload:

{
  "metadata": {
    "description": "This is a cat",
    "image": "https://i.imgur.com/4M34hi2.jpeg"
  },
  "clientId": "your-client-id"
}

Response​

Success response with the updated guild object.

{
  "status": "Success",
  "data": {
    // The updated Guilds object will be returned here
  }
}

Update Maximum Number of Members for a Guild​

Endpoint​

PUT /max-members/:clientId/:guildId/:maxMembers

Authorization​

  • Guarded by ApiKeyGuard
  • Requires Api key (api-key)

Headers​

  • api-key: <api_key>

Description​

This endpoint is used to update the maximum number of members that a guild can have. It requires the client ID, guild ID, and the new maximum number of members as URL parameters.

URL Parameters​

  • clientId (string): The identifier of the client making the request.
  • guildId (string): The unique identifier of the guild.
  • maxMembers (number): The new maximum number of members to be set for the guild.

Response​

200 OK​

Indicates that the maximum number of members for the guild has been successfully updated.

{
  "status": "Success",
  "message": "The maximum number of members has been updated successfully."
}

Ban User by User ID (from Game Server)​

Endpoint​

PUT /ban-user/:clientId/:guildId/:userId

Authorization​

  • Guarded by ApiKeyGuard
  • Requires Api key (api-key)

Headers​

  • api-key: <api_key>

Description​

This endpoint is used to ban a user from a guild based on their user ID. It requires the client ID, guild ID, and the user ID of the user as URL parameters.

URL Parameters​

  • clientId (string): The identifier of the client making the request.
  • guildId (string): The unique identifier of the guild.
  • userId (string): The user ID of the user who is to be banned.

Response​

200 OK​

The successful response indicates that the user is banned from the guild.

{
  "status": "Success",
  "message": "User banned successfully."
}

Get All Join Guild Requests of Game​

Endpoint​

GET /join-guild-request/game/:clientId

Authorization​

  • Guarded by ApiKeyGuard
  • Requires Api key (api-key)

Headers​

  • api-key: <api_key>

Description​

This endpoint retrieves all join-guild requests of a user identified by the clientId.

Request Parameters​

  • clientId: The identifier of the user.

Query Parameters​

The request can include optional query parameters:

  • page: The page number for pagination (default is 1).
  • limit: The maximum number of join guild requests to retrieve per page (default is 10).
  • userId: The id of user
  • guildId: The id of the guild

Response​

Returns a list of join guild requests for the specified user.

Example response:

[
  {
    "_id": "string",
    "clientId": "string",
    "userId": "string",
    "guild": "string",
    "status": "string",
    "updatedAt": "string"
  }
]

Accept Join Guild Request​

Endpoint​

PUT /join-guild-request/accept

Authorization​

  • Guarded by ApiKeyGuard
  • Requires Api key (api-key)

Headers​

  • api-key: <api_key>

Description​

This endpoint is used to accept a join guild request.

Request Body​

The request body must be a JSON object following the structure defined by AcceptJoinGuildDto.

  • clientId: The id of client
  • userId: The id of user
  • guildId: The id of the guild

Response​

Returns the accepted join guild request.

Example response:

 {
    "_id": "string",
    "clientId": "string",
    "userId": "string",
    "guild": "string",
    "status": "string",
    "updatedAt": "string"
  },
