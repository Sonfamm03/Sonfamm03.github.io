How SDK works

  • After the user authentication stage, partner can access to the Properties and Methods in ShardsTech SDK.

  • There are some methods that need to call blockchain in our SDK: CreateGuild, UpdateGuild, BuySeat, BuyShare, SellShare, LinkAddress. Partner should have an UI Processing and pass CancellationToken into mentioned methods.

    • User account will automaticaly link to an address which is not used by other account after executing any method. Therefore, LinkAddress dosen't need to be called.

    • We provide 2 options to call these methods:

      • Call through MiraiApp:
        • Good: Risk-free, easy to implement, many features to protect users
        • Bad: Inefficient UX, need to use 2 apps
        • How to call:
          • If user's device has MiraiApp already, SDK will call directly through deeplink miraiapp:// and callback to you game through your AppSchema setting before (Example: shardstechdemo://)
          • Or else we provide 2 options to call: Universal Link or SendNotification
            • Universal Link: If your device is Android/IOS will redirect to Google Play / App Store to install MiraiApp. Else if Unity Editor or Window, Mac, WebGL will open Universal Link in browser and show QR code, please use MiraiApp scan it to execute action
            • SendNotification We try send notification to MiraiApp by user address, open notification to execute action. For case Account isn't linked to any address will fallback to Universal Link
      • Use Embedded Wallet (Development):
        • Good: Only 1 app is need for all features
        • Bad: More risk (policy, security), implement more things and more complex.
